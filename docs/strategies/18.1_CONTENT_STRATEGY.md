# MS18 Phase 1 Strategy — Content Strategy, Helpful Notes Audit & CMS Selection

Status: Active.
Parent milestone doc: [`docs/milestones/18_PRERELEASE_CONTENT_SEO.md`](../milestones/18_PRERELEASE_CONTENT_SEO.md)

**Goal:** Audit the existing Helpful Notes as raw material, select and set up a lightweight CMS, and produce an approved content plan — so Phase 2 inherits a working publishing foundation and Phase 3 can go straight to writing.

## 1. Helpful Notes audit
1. [ ] Read each Helpful Note in `resources/views/help/index.blade.php` and record an Adapt / Inspire / Skip decision with rationale.
2. [ ] Confirm or revise the preliminary decisions already recorded in the reference table below.
3. [ ] Flag any notes where the general concept is strong but the content needs substantial rewriting before it can stand alone.

## 2. CMS selection and basic setup
1. [ ] Evaluate lightweight CMS options for a GitHub Pages static site (Jekyll, Hugo, raw HTML). Recommended starting point: Jekyll (natively supported, no build step, Markdown authoring).
2. [ ] Select a CMS and document the decision with rationale.
3. [ ] Perform basic setup: config file, base layout, convert `site/index.html` to use the layout.
4. [ ] Confirm GitHub Pages still builds and serves correctly after setup.
5. [ ] Confirm a new Markdown content file produces a working page before closing Phase 1.

## 3. Original article topics
1. [ ] Confirm or revise the preliminary original article topics listed in the reference section below.
2. [ ] Add or remove topics based on Helpful Notes audit findings.

## 4. Content plan
1. [ ] Produce a prioritized ordered list of all content to produce in Phase 3: adapted notes, original articles, and any video scope.
2. [ ] Get content plan approved before Phase 2 begins.

## Exit Criteria
- [ ] Every Helpful Note has a confirmed Adapt / Inspire / Skip decision.
- [ ] CMS selected, set up, and confirmed working on GitHub Pages.
- [ ] Original article topics confirmed.
- [ ] Prioritized content plan approved.
