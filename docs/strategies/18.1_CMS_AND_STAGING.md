# MS18 Phase 1 Strategy — CMS Selection & Staging Setup

Status: Active.
Parent milestone doc: [`docs/milestones/18_PRERELEASE_CONTENT_SEO.md`](../milestones/18_PRERELEASE_CONTENT_SEO.md)

**Goal:** Select a CMS and staging workflow as joint decisions, set both up, and confirm the full publish pipeline works end-to-end — so Phase 2 can go straight to writing without tooling uncertainty.

## 1. Evaluate CMS and staging options together
1. [ ] List candidate CMS options (Jekyll, Hugo, raw HTML) with their staging implications.
2. [ ] List candidate staging approaches (branch-based GitHub Pages preview, local `serve` only, external preview service) with their CMS implications.
3. [ ] Identify any combinations that are ruled out by the other decision.

## 2. Make the joint decision
1. [ ] Select a CMS. Document the choice and rationale.
2. [ ] Select a staging approach. Document the choice and rationale.
3. [ ] Confirm the two choices are compatible and reinforce each other.

## 3. CMS setup
1. [ ] Add CMS config file and base layout to `site/`.
2. [ ] Convert `site/index.html` to use the base layout.
3. [ ] Confirm the existing placeholder page still renders correctly after conversion.
4. [ ] Confirm GitHub Pages builds and serves correctly.

## 4. Staging setup
1. [ ] Implement the chosen staging workflow.
2. [ ] Create a test content page and run it through the full staging workflow.
3. [ ] Confirm the test page is reviewable in staging before it touches the live site.
4. [ ] Confirm the test page goes live correctly when promoted.
5. [ ] Remove or archive the test page after verification.

## Exit Criteria
- [ ] CMS and staging approach selected and documented.
- [ ] CMS set up; existing placeholder page unaffected.
- [ ] Staging workflow set up and verified end-to-end.
- [ ] Phase 2 can begin writing content immediately without tooling work.
