# MS17 Phase 5 Strategy - Post-Payment Onboarding

Status: Pending.
Parent milestone: [`docs/milestones/17_PRODUCT_READINESS.md`](../milestones/17_PRODUCT_READINESS.md)
Depends on: Phase 3 (off-chain payments) complete.
Canonical inputs: [`docs/specs/NOTIFICATIONS.md`](../specs/NOTIFICATIONS.md), [`docs/PRODUCT_SPEC.md`](../PRODUCT_SPEC.md)

The receipt send path already exists. The gap is that issuers don't always know it's waiting for them. This phase ensures that when an invoice reaches paid state without a sent client receipt, the app surfaces a clear, timely prompt so the issuer doesn't miss the follow-through step. It also audits the issuer paid notice email to confirm it orients the issuer toward that next step without being directive.

## 1. Audit the current post-payment issuer experience

Identify where the receipt send path is surfaced and where it is missing or easy to overlook.

1. [ ] Review the invoice show view receipt review panel — is it prominent enough on a freshly paid invoice? Does it appear correctly regardless of how the invoice reached paid state (on-chain, manual payment, small-balance resolver)?
2. [ ] Review the dashboard — is the "Review receipt" prompt shown for all paid invoices without a queued or sent receipt?
3. [ ] Review the issuer paid notice email — does it orient the issuer toward the receipt review step clearly and without overreaching?
4. [ ] Document findings before making any changes.

## 2. Improve post-payment surfaces

Address gaps identified in section 1.

1. [ ] Invoice show view: ensure the receipt review panel is visible and appropriately prominent for any paid invoice without a sent receipt, regardless of how paid state was reached.
2. [ ] Dashboard: ensure the "Review receipt" prompt appears consistently for all qualifying invoices, including those paid via manual payment (Phase 3).
3. [ ] Issuer paid notice email: if the audit finds the copy doesn't orient the issuer toward the receipt review step, adjust it. Keep copy honest — this is an orientation nudge, not a directive.

## 3. Verify Phase 5

1. [ ] An invoice paid via on-chain payment shows the receipt review panel and dashboard prompt correctly.
2. [ ] An invoice paid via manual payment (Phase 3) shows the receipt review panel and dashboard prompt correctly.
3. [ ] The issuer paid notice email orients the issuer toward the receipt review step.
4. [ ] No automatic receipt is sent — the manual review gate remains intact.
5. [ ] Browser QA: walk the full post-payment flow from detection through receipt send for both on-chain and manual payment paths.
